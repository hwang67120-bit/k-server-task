# 기술 선택 기록 템플릿

락, 트랜잭션, 캐시, Kafka, Outbox 같은 기술과 설계 전략을 결정할 때 사용한다.

## 선택 기록

### 해결할 문제

현재 구조에서 어떤 구체적인 문제가 발생하는가?

### 발생 조건

어떤 요청량, 동시 실행, 장애 또는 데이터 조건에서 문제가 발생하는가?

### 비즈니스 영향

어떤 사용자 시나리오, 비즈니스 정책 또는 데이터 불변식이 깨질 수 있는가?

### 검토한 방법

단순한 방법과 복잡한 방법을 포함해 어떤 대안을 검토했는가?

### 선택한 방법

어떤 기술 또는 전략을 선택했는가?

### 선택 이유

현재 요구사항과 환경에서 왜 이 방법이 적절한가?

### 비용과 트레이드오프

어떤 구현 복잡도, 운영 비용, 성능 비용 또는 일관성 문제가 추가되는가?

### 검증 방법

어떤 테스트, 로그, 지표 또는 적용 전후 측정으로 효과를 증명하는가?

### 재검토 조건

어떤 측정값 또는 요구사항 변화가 발생하면 다른 방법을 다시 비교하는가?
